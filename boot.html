Explaining the boot process

Basic boot sequence:

  1. BIOS/UEFI locates and executes boot program, or boot loader.
  2. boot loader loads the kernel
  3. kernel starts the init process (pid=1)
  4. init manages system initialization, using SysVinit startup scripts, or using Upstart or systemd.

When the computer is powered on it can only perfom operations that it is ordered to do by the BIOS.

The BIOS runs the POST (Power On Self Test). This checks memory and hardware then searches a specific location or device for a boot program.

The boot program is usually found in the MBR (Master Boot Record). Control is then passed to this boot program (usually GRUB).

The boot program loads the kernel into the memory and executes it. On x86 platforms (and others) it decompress itself. Then performs hardware checks, gains access to peripheral hardware, and runs the init process.

The init process continues the system startup, running init scripts if SysVinit is being used, or managing Upstart or systemd.

The BIOS is uusally placed in a ROM chip (ROM BIOS) that comes with the computer. This ensures that the BIOS will always be available and will not be damaged by disk failures, and makes it possible for a computer to boot itself.

During the boot process the BIOS loads the boot loader from the MBR.
